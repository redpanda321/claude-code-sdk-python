# Contributing to claude-code-sdk

Thanks for your interest! This is an Alpha-stage Python port of the
Claude Code Builder TypeScript package. Contributions are welcome via
issues and pull requests.

## Dev setup

Requirements: Python 3.11+ and [uv](https://docs.astral.sh/uv/).

```bash
git clone https://github.com/redpanda321/claude-code-sdk-python
cd claude-code-sdk-python
uv sync --extra dev --extra agent-sdk
```

## Run checks locally

All checks must pass before a PR lands:

```bash
uv run ruff check src/ tests/
uv run ruff format --check src/ tests/
uv run pytest tests/ -v
uv run pyright src/ tests/   # strict mode, currently soft-fail in CI
```

Auto-format:

```bash
uv run ruff format src/ tests/
```

## Commit style

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat(scope): ...` -- new user-visible behaviour
- `fix(scope): ...` -- bug fix
- `docs(scope): ...` -- README/CHANGELOG/etc.
- `test(scope): ...` -- tests only
- `refactor(scope): ...` -- internal refactor, no behaviour change
- `ci(scope): ...` -- CI/workflow changes
- `style(scope): ...` -- formatter/lint-only changes

Scope is typically the subpackage (`tools`, `adapters`, `mcp`, ...) or a
phase identifier (`999.2-05`).

## Pull request checklist

- [ ] Tests added / updated for new behaviour
- [ ] `ruff check` and `ruff format --check` clean
- [ ] `pytest` green on 3.11 / 3.12 / 3.13 (CI runs the matrix)
- [ ] No `[project.scripts]` added to `pyproject.toml` -- **this package is
      library-only**; the CLI lives in a separate repo (PY-SDK-04).
- [ ] Public API changes mentioned in `CHANGELOG.md` under `## [Unreleased]`

## Release process (maintainers only)

Versions are derived from git tags via `hatch-vcs`; there is no `__version__`
bump to commit.

```bash
# 1. Ensure main is green
gh run list --workflow=ci.yml --limit 1

# 2. Tag + push
git tag v0.1.0a1
git push origin v0.1.0a1

# The release.yml workflow publishes via PyPI trusted publishing (OIDC).
# Alternatively, manually dispatch:
gh workflow run Release --ref main -f dry-run=false
```

The `pypi` GitHub Environment must be configured as a trusted publisher on
PyPI before the first release.

```
  adapters/                 # populated: claude-agent-sdk bridge
  <11 other barrels>/       # placeholder __all__ = [] with CCB-drain docstring

```
